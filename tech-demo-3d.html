IRONQUAD Tech Demo

Procedural quadruped mech driven by physics-based movement

Project Overview

This project is ongoing; the features presented here represent the current stage of development, with additional mechanics and refinements planned.

IRONQUAD Tech Demo3D is a Unity technical sandbox exploring a procedural quadruped mech powered by Rigidbody-based locomotion and procedural inverse kinematics, rather than animation clips. The focus is grounded movement, gait control, and stable foot placement using raycasts and a procedural leg rig.

The core loop is: the body moves via physics, each leg computes an “ideal” foot target by raycasting against the terrain, and the system triggers a step when the current foot position drifts beyond a threshold—moving the foot along a smooth arc for mechanically grounded motion.

Details & Responsibilities

Locomotion & Physics

Built a leg control pipeline (ideal foot target → step decision → procedural step motion → IK solve) and implemented Rigidbody-driven movement with hover/jetpack behavior for stability across uneven terrain.

Core Mechanics
  • Procedural Stepping (Real-Time Generation): Foot motion is generated at runtime using eased interpolation and a sine-based lift arc.
  • Grounding via Raycasts: Each leg maintains a surface-adapted target snapped to the ground to define ideal foot placement.
  • Gait & Step Trigger Logic: Steps trigger when displacement exceeds a threshold (e.g., moveDistance), controlling cadence and stability.

Procedural Animation

Designed stepping arcs using SmoothStep easing with a sine-wave lift to simulate weight transfer, and implemented visual leg joints that bend toward the computed foot target for responsive mechanical articulation.

Implementation Details

Step Arc Logic

Generates a smooth arc using easing and trigonometry:

Vector3 CalculateFootPos(Vector3 start, Vector3 end, float t) { float smoothT = Mathf.SmoothStep(0f, 1f, t); float height = Mathf.Sin(t * Mathf.PI) * stepHeight; return Vector3.Lerp(start, end, smoothT) + Vector3.up * height; }

Game Systems & Controls

Implemented auxiliary systems including obstacle avoidance, player controls, and debug tools.

Controls
  • WASD: Physics-based movement
  • Space: Hover / Jetpack
  • Auto-Hover: Triggers on wall detection
Debug & Iteration
  • Tuned thresholds (step distance, step height) and validated behavior with gizmos/debug visualization.
